# 02 - 機能要件・非機能要件

## 機能要件

### 認証

- [ ] メールアドレスによるマジックリンク（OTP）でのログイン・ログアウト
- [ ] 各ユーザーは自分のデータのみ閲覧・編集可能
- [ ] 未認証時でもアプリを体験可能（ローカルストレージにデータを保存）
- [ ] `/settings` のみ認証必須。その他のページは認証不要でアクセス可能
- [ ] ログイン後はローカルストレージのデータを自動的にSupabaseへ移行する

### プロジェクト管理

- [ ] プロジェクトの作成・編集・削除（CRUD）
- [ ] プロジェクトに「タイトル」「期日」を設定
- [ ] 複数プロジェクトを並行管理
- [ ] ダッシュボードにプロジェクト一覧をリスト形式で表示

### タスク入力

- [ ] テキストエリア形式のインデント入力UI
- [ ] Tabキーで1段階インデントを深める（子タスク化）
- [ ] Shift+Tabキーで1段階インデントを浅くする（親タスクに昇格）
- [ ] 階層は無制限
- [ ] タスク名末尾に `[Nd]`（例: `[3d]`）形式で所要日数を指定可能
- [ ] 最下位階層のタスク（子タスクなし）は `[Nh]`（例: `[4h]`）形式で所要時間を指定可能（1日=8時間）
- [ ] 親タスクに `[Nh]` が指定された場合は無視してデフォルト（1日）として扱う
- [ ] 所要日数・時間を省略した場合は1営業日として扱う
- [ ] 「生成」ボタン押下でガントチャートに反映（リアルタイムではなく手動）

### タスクが持つ情報

- [ ] タイトル
- [ ] 開始日 / 終了日
- [ ] ステータス（未着手 / 進行中 / 完了）
- [ ] 進捗率（0〜100%）
- [ ] メモ・備考欄

### スケジュール自動計算

- [ ] プロジェクト期日から逆算してタスクを自動配置
- [ ] 同一階層内は上から順に直列（前タスク完了後に次タスク開始）
- [ ] 土日をスキップ（祝日は考慮しない）
- [ ] タスク総日数が期日に収まらない場合、各タスクの日数を比例圧縮して期日内に収める
- [ ] 圧縮発生時はUI上でユーザーに警告表示

### ガントチャート

- [ ] frappe-ganttによるガントチャート表示
- [ ] 時間単位タスク（`[Nh]`）はガントチャート上では最低1日のバーで表示し、ホバー時にツールチップで所要時間（例: 「所要時間: 4時間」）を表示
- [ ] 期日までの日数で表示スケールを自動切替
  - 〜30日: 日単位
  - 31〜90日: 週単位
  - 91日〜: 月単位
- [ ] 手動でスケールを切り替えるUIボタンを提供
- [ ] ガントチャート上でバーをドラッグして日程を手動変更
- [ ] ドラッグ後、後続タスクが連鎖してずれる（Phase 2）

### 通知

- [ ] 各タスクの開始日前日にアプリ内バナー通知を表示
- [ ] バナーはページ上部に常設表示（開始が迫っているタスク一覧）
- [ ] バナーを閉じた場合は当日セッション中は再表示しない

### 画面構成

- [ ] `/login` - ログイン画面（メールアドレス入力 → マジックリンク送信）
- [ ] `/dashboard` - プロジェクト一覧（リスト形式）
- [ ] `/projects/new` - プロジェクト作成フォーム
- [ ] `/projects/:id` - タスク入力タブ + ガントチャートタブ（切り替え式）
- [ ] `/settings` - ユーザー設定

---

## 非機能要件

### パフォーマンス

- 初回ロード（キャッシュなし）: 3秒以内
- ガントチャートの生成: 「生成」ボタン押下後1秒以内に描画

### セキュリティ

- Supabase Row Level Security（RLS）ポリシーにより、ユーザーは自分のデータのみ読み書き可能
- 認証トークンはSupabase Authが管理（独自実装なし）
- 環境変数（`.env`）を`.gitignore`に含め、リポジトリに含めない

### 可用性・ホスティング

- Vercelによるホスティング（SPA対応）
- Supabaseの無料枠（Free tier）内での運用を想定（DB: 500MB、月間アクティブユーザー: 50,000件）

### 対応ブラウザ

- Chrome / Firefox / Safari 最新版（PCのみ）

### コード品質

- TypeScriptを使用し型安全を確保
- ESLint + Prettier による静的解析・フォーマット統一
