# 06 - 画面構成・UI/UXデザイン方針

## 画面一覧とルーティング

| ルート | 画面名 | 認証 |
|---|---|---|
| `/login` | ログイン画面 | 不要 |
| `/dashboard` | プロジェクト一覧 | 不要（ローカルモードで利用可） |
| `/projects/new` | プロジェクト作成 | 不要（ローカルモードで利用可） |
| `/projects/:id` | プロジェクト詳細 | 不要（ローカルモードで利用可） |
| `/settings` | ユーザー設定 | 必須 |

---

## 各画面の詳細

### /login - ログイン画面

- 中央にアプリロゴ + メールアドレス入力フォーム + 「マジックリンクを送信」ボタン
- 送信後は「メールを確認してください」メッセージを表示
- ログイン済みの場合は `/dashboard` にリダイレクト
- フォームの下に「ログインせずに体験する →」リンクを表示し、`/dashboard` へ遷移

---

### /dashboard - プロジェクト一覧

#### レイアウト

- ヘッダー: アプリ名 + ユーザーアイコン（ログイン時: ドロップダウンでログアウト / 未ログイン時: 「ログイン」ボタン）
- ローカルモードバナー: 未ログイン時にのみ表示（詳細は後述）
- 上部バナー: 開始が迫っているタスク通知（詳細は後述）
- 「新規プロジェクト作成」ボタン
- プロジェクトをリスト形式で表示

#### プロジェクトリスト（1行の表示内容）

```
| プロジェクト名    | 期日       | 進捗率 | ステータス | 操作       |
|----------------|------------|--------|----------|-----------|
| 引越し          | 2026/04/30 | 40%    | 進行中   | 開く / 削除 |
| 学術発表        | 2026/06/15 | 10%    | 未着手   | 開く / 削除 |
```

- プロジェクト行クリック or「開く」で `/projects/:id` へ遷移
- 進捗率はタスク全体の平均（完了タスク数 / 全タスク数 × 100）

---

### /projects/new - プロジェクト作成

- フォーム: プロジェクト名（必須）/ 期日（必須）
- 「作成」ボタンで Supabase に保存後、`/projects/:id` へ遷移

---

### /projects/:id - プロジェクト詳細

#### タブ構成

ページ上部にタブを配置し、2つのビューを切り替える。

```
[ タスクリスト ] [ ガントチャート ]
```

#### タブ1: タスクリスト

**インデント入力エリア**

```
┌─────────────────────────────────────────────────┐
│ 引越し作業                                        │
│   不用品処分 [2d]                                 │
│   荷物梱包                                        │
│     リビング                                      │
│     寝室                                         │
│   引越し当日                                      │
│                                                  │
└─────────────────────────────────────────────────┘
                                      [ 生成 ]
```

- 操作:
  - `Tab`: インデントを1段深める
  - `Shift+Tab`: インデントを1段浅くする
  - タスク名末尾に `[Nd]` で日数指定（例: `[3d]`）
  - 最下位タスク（子なし）は `[Nh]` で時間指定も可能（例: `[4h]`、1日=8時間）
  - 「生成」ボタン押下でパース → スケジュール計算 → ガントチャートタブへ反映

**生成後のタスク詳細リスト**

生成後は各タスクを展開可能なリストで表示。タスク行をクリックすると詳細パネルが開く。

詳細パネルの項目:
- タイトル（編集可）
- 開始日 / 終了日（読み取り専用。ガントチャートから変更）
- ステータス（プルダウン: 未着手 / 進行中 / 完了）
- 進捗率（スライダー or 数値入力）
- メモ（テキストエリア）

#### タブ2: ガントチャート

```
┌──────────────────────────────────────────────────────────┐
│ スケール: [日] [週] [月]                  今日: 2026/03/07 │
├────────────────┬─────────────────────────────────────────┤
│ 引越し作業     │ ══════════════════════════════          │
│   不用品処分   │ ████                                    │
│   荷物梱包     │     ████████                            │
│     リビング   │     ████                                │
│     寝室       │         ████                            │
│   引越し当日   │                 ████                    │
└────────────────┴─────────────────────────────────────────┘
```

- バーをドラッグして日程変更可能（Phase 2）
- ドラッグ後、後続タスクが連鎖してずれる（Phase 2）
- スケール切替ボタン（日 / 週 / 月）
- 時間単位タスク（`[Nh]`）はガントチャート上で最低1日のバーで表示し、ホバー時に「所要時間: N時間」とツールチップで表示

### /settings - ユーザー設定

- ユーザー情報の表示（名前・メール）
- ログアウトボタン
- （将来）通知設定

## ローカルモードバナー

### 表示条件

- 未ログイン状態でダッシュボード・プロジェクト画面を表示している場合

### 表示位置

- ページ最上部（ヘッダーの直下）に固定バナー（通知バナーより上位）

### デザイン（案）

```
┌─────────────────────────────────────────────────────── [×] ┐
│  現在はローカル保存モードです。ログインするとクラウドに保存され、 │
│  複数デバイスからアクセスできます。             [ログイン]  │
└─────────────────────────────────────────────────────────────┘
```

- [×] ボタンで当日のセッション中は非表示（`sessionStorage` 利用）
- 「ログイン」ボタンで `/login` へ遷移

---

## アプリ内バナー通知

### 表示条件

- 現在日付が「タスク開始日の前日」であるタスクが存在する場合

### 表示位置

- ページ最上部（ヘッダーの下）に固定バナー

### デザイン（案）

```
┌──────────────────────────────────────────────────── [×] ┐
│  明日開始のタスクがあります: 「リビング梱包」（引越しプロジェクト）  │
└────────────────────────────────────────────────────────┘
```

- 複数ある場合はまとめて1つのバナーに表示（「他N件」）
- [×] ボタンで当日のセッション中は非表示（`sessionStorage` 利用）

---

## UIデザイン方針

- カラーパレット: ニュートラル系（グレー基調）＋ アクセントカラー1色
- フォント: システムフォント（Tailwind のデフォルト）
- レスポンシブ: PC（1280px〜）を主軸。それ以下はレイアウト崩れを許容（将来対応）
- アニメーション: 最小限（タブ切り替え時のフェードのみ）
